# 算法 1：位运算

数据范围很具有迷惑性，一度让我以为是个容斥 + 解线性同余方程…… 结果居然是个大力 bitset 暴力题。

开一个大小为 10910^9109 的 bitset ，然后对于 SSS 中的每一个元素，都暴力枚举它的倍数，在 bitset 中标记倍数。那么答案就是 bitset 中连续 333 个 111 的个数（这可以通过 tmp & (tmp >> 1) & (tmp >> 2) 并统计其中 111 的个数计算，想想为什么）。

这么做的时间复杂度是 O(n∣S∣64)O\left(\frac{n\left|S\right|}{64}\right)O(64n∣S∣​) 的。除此之外，我们可以考虑手写 bitset 来进一步优化常数（开成一个 unsigned long long 数组并在其中模拟 bitset 操作），但是复杂度还是有点高，下面我们来进行一个很重要的优化：

注意到在 sis_isi​ 很小的时候我们会枚举很多次它的倍数，但是容易发现，sis_isi​ 筛出来的 010101 串是具有循环节的，且循环节长度是不超过 xxx 的。之后我们直接拿这个循环节去标记 unsigned long long 数组就好了。在这里我给 sis_isi​ 设置的阈值是 646464，如果小于 646464 就找循环节，否则大力枚举倍数标记。

还是应该关注一下代码实现，写完这道感觉对位运算的理解上升到了新的高度
