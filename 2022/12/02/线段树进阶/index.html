# 二分区间

有些时候，题目要求我们找的区间可能有很多个符合给定性质，然后我们要找出第一个 / 最后一个符合条件的区间。这个时候我们通常会考虑加强性质使得满足加强性质的区间只有一个（通常来说，取最值），然后在线段树上维护这个区间，并以这个 “代表区间” 为基准进行二分。

举个例子，题目要我们在序列中找到第一个长度 ≥x\ge x≥x 的区间符合某种性质。考虑加强条件：找到最长的符合该性质的区间，并在线段树上维护这个区间的长度，以此作为代表区间。然后进行二分：因为要找到第一个，所以我们先递归左半边，如果有，返回答案；然后找跨终点区间；最后递归右半边。

可以发现，这种二分的问题通常会和子序列线段树结合。一般来说使用递归版的二分更好写。

# 二分单点，有单调性

在序列上找到第一个满足某个性质的点，类比二分区间的思路，但这次我们不需要子序列线段树，也不需要考虑跨中点，只需要考虑左边有没有，然后再考虑右边有没有。一般来说使用循环版的二分更好写。

在使用线段树的循环版二分的时候，建议往树的右边插入一个永远不可能取到的点，防止线段树上原本最靠右的点在二分过程中不被考虑到。

# 二分单点，无单调性

一般的二分都需要保证单调性，但是由于线段树的特殊性，我们可以快速得到一个线段树节点对应区间的信息，据此我们可以判断是否有一个答案落在这个范围内。但是注意：我们只能说答案有可能落在这里面，但是答案不一定就在，我们还得递归下去看是不是真的在，这也就要求我们通过一定手段，能让我们递归下去的次数满足时间复杂度要求，否则就会把整棵线段树都遍历一遍，时间复杂度爆炸。

此时的线段树二分已经和传统二分用线段树优化完全不一样了，当然也和上面两种完全不一样。或者说，这是在这棵树上搜索罢了。

# 势能分析线段树（Seg-Beat）

有些区间修改操作并不方便我们用懒标记和快速更新的方式维护（比如区间开放、区间取模），但是我们发现，如果不断对这个区间做这种操作，到了一定次数之后，即使再做出修改，这个区间也不会有改变了，此时，我们可以直接暴力更新而不用担心事件复杂度高。

之所以称为势能分析，是因为这种操作很类似于物理里 “势能” 的概念，一个物体在一定高度，它所具备的重力势能就是这么多，重力只能做这么多的功，而和做功过程是无关的。类似地，比如对于区间开方操作，如果只有这种操作，那么一个数不久之后就会开成 111，之后再开方就已经没有意义了，可以不管。这就是 势能分析 的思路，某种意义上只是在考察我们时间复杂度的分析能力。

# 辅助数组线段树（出现次数问题）

如果一个点的权值还和它的出现次数有关（这里以第一次出现才计权值举例），我们会维护额外数组来辅助线段树统计权值（比如记录 preipre_iprei​ 表示 iii 前一次出现的位置，对于询问 [l,r]\left[l,r\right][l,r] 和位于区间内的 iii，只有 prei≤l−1pre_i\le l-1prei​≤l−1 才能记录权值）。

如果不需要强制在线，那么可以把询问离线下来按照右端点排序，用普通线段树维护；如果强制在线，那么我们就进行预处理，使用可持久化线段树模拟一遍右端点的增长过程，然后在一一处理询问。

这似乎是一个很经典的模型，这里有好几道例题都要灵活应用这个 preprepre 数组。

# 分治线段树

不是线段树分治！！！

遇到询问有多少个区间满足某种性质的问题，我们考虑用分治解决。

线段树本身就是分治思想，将两种算法结合到一起再自然不过了。不过一般分治的时间复杂度通常不足以保证能多次询问，所以我们还要进一步优化我们计算跨中点情况的算法，这样才能借助线段树来做到多次询问甚至动态修改。最终分治过程会写进线段树的 pushup 函数里。
