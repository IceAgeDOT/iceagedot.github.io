# 算法 1：分治 + 线段树

对于询问有多少个区间满足某种性质的问题，我们先考虑一个朴素分治。

[l,mid]\left[l,mid\right][l,mid] 和 [mid+1,r]\left[mid+1,r\right][mid+1,r] 递归求解。

[l,r]\left[l,r\right][l,r] 的区间需要通过 [l,mid]\left[l,mid\right][l,mid] 的后缀和 [mid+1,r]\left[mid+1,r\right][mid+1,r] 的前缀来求解。

注意到按位或的性质，或的数越多，或的结果只增不减，因此这里可以用双指针来统计跨过中点的询问。

这样单次询问的复杂度是 O(nlog⁡n)O\left(n\log n\right)O(nlogn) 但是只能支持一次询问（雾）。

多次询问显然考虑到用线段树，而分治就是天然的划分线段树的区间，我们要优化上述分治算法，作为线段树的 pushup\text{pushup}pushup 函数。

但是我们发现由于按位或的性质，前缀 / 后缀或的结果不会变化超过 O(log⁡a)O\left(\log a\right)O(loga) 次。所以我们考虑线段树在每个节点记录一下是哪些位置让这个发生了改变。

我们记录一个 vector prefix\text{vector prefix}vector prefix 表示区间 [l,r]\left[l,r\right][l,r] 从 lll 开始计算，会变化的位置以及变化的值。

同理我们记录一个 vector suffix\text{vector suffix}vector suffix 表示区间 [l,r]\left[l,r\right][l,r] 从 rrr 开始计算，会变化的位置以及变化的值。

显然这个信息是可以在线段树上面合并的，每次 pushup\text{pushup}pushup 当前区间的复杂度也只需要 O(log⁡n)O\left(\log n\right)O(logn)。

然后这题就做完了，总的时间复杂度为 O(nlog⁡2n)O\left(n\log^2n\right)O(nlog2n)。

注意实现的时候细节有亿点多，所以推荐边写边画图，方便考虑清楚所有边界条件。
