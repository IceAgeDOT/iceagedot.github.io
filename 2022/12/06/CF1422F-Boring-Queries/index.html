# 算法 1：可持久化线段树 + 根号分治

首先肯定是想到最简单的思路：用线段树维护区间 lcm\text{lcm}lcm。不幸的是，高精度会超时，取模也无法维护（我感觉似乎是可以通过维护 gcd\text{gcd}gcd 和区间乘积的但是我还没尝试）。

这个时候考虑把 lcm\text{lcm}lcm 的计算公式写出来：∏i=1kpimax⁡(αi)\prod\limits_{i=1}^kp_i^{\max\left(\alpha_i\right)}i=1∏k​pimax(αi​)​。其中 pip_ipi​ 是质数，它的指数是所有数分解质因数后最大的那个指数。也就是说，我们可以通过维护 区间最大指数 来解决这个问题。

但是我们不能维护很多质因数，因为 nnn 以内的质数大约有 O(nln⁡n)O\left(\frac{n}{\ln n}\right)O(lnnn​) 个。这个时候有个常规套路：按照 n\sqrt nn​ 分成两部分，前半部分的质数的指数可以有很多次但是数量不多，后半部分的质数很多但是最多出现 111 次。

解决前半部分很简单，用多棵线段树 / ST\text{ST}ST 表维护区间最大值即可。关键是后面怎么维护。

对于一个区间，可能出现很多个大质数，但是我们只能统计一次，也就是说 出现次数和权值相关。这是一个经典的模型，我们通过将询问右端点离线，每次多考虑一个右端点就把右侧的质数算上，再把上一次出现的消除权值即可。

但是这道题要求强制在线，所以我们可以把询问离线变成可持久化线段树来解决。
